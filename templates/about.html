Introduction

EVE has no great way to monitor structures for organizations that do not fit in the Alliance / Corp structure of EVE.

This site attemtps to do that.

Logging In

This site uses EVE Online Single Sign On.

The CCP "Help & Support" site has a page that shows which third party applications that you have granted permissions to. This site is the "Structures and Minerals" entry.

Almost everyone will want to login as a User. This enables viewing of structure information, and requires minimal data to login.

If you manage structures, and want to contribute your structure information, then login as a Contributor. You will need the in-game corporate roles of "Station Manager" and "Accountant" (see this article for more information about corporation roles).

If you are not sure how you are logged in, look at your character image at the top of the window. If your character has a thin border the color of the foreground - you are logged in as a User. If your character has a thicker border in yellow - you are logged in as a Contributor.

If you want to login with more than one character at the same time, then you can visit the user and contributor login links directly.

Permissions

Because all EVE players trust each other entirely, ... this site has a permissions system based on Alliances / Corporations / Users.

CCP does not (currently) have a way to access in-game ACL information from outside-of-game. So the site has no way of knowing if you are included / excluded in-game.

If you get a "Permission Denied" page in error, please contact Jay Blunt in-game to get the permissions fixed.

Logging Out

This site will try and maintain your login session. If you change IP address or some other Internet voodooo, it will assume you have logged out.

If you navigate away from ths site without explictly logging out .. the site will try and remember you for next time.

If you click your character image, you will be logged out.

If you are logged out, you will be served the login page when you visit.

Technical Info

This site is mostly async python with aiohttp and sqlalchemy.

The server is a www.vultr.com virtual server. It is running OpenBSD. The framework used is Quart. The TLS reverse proxy is OpenBSD's relayd.
